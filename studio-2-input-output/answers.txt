10. Changing bufferSize does not affect correctness as long as it is greater than 0. Even if it is very small, the program still works correctly, but it reads and writes smaller chunks at a time.

11. A small bufferSize causes more system calls, which increases overhead and reduces performance but uses less memory. A large bufferSize reduces the number of system calls and improves performance but uses more memory. The tradeoff is between efficiency and memory usage.
