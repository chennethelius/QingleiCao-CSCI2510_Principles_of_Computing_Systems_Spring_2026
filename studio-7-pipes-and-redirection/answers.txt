3)

- fd[0] contains the read end of the pipe.
- fd[1] contains the write end of the pipe.

4)
newfd is closed when dup2() succeeds.
So for our case:
- dup2(fd[1], STDOUT_FILENO) closes STDOUT_FILENO and replaces it with fd[1]
- dup2(fd[0], STDIN_FILENO) closes STDIN_FILENO and replaces it with fd[0]

5)
In child 1 (before exec'ing Program 1):
    dup2(fd[1], STDOUT_FILENO);

In child 2 (before exec'ing Program 2):
    dup2(fd[0], STDIN_FILENO);

6)
Output (program still doesn't terminate without close() calls):
Program 2 got: Hello from Program 1!

7)
Added close() calls:
- Parent closes both fd[0] and fd[1] (doesn't use the pipe)
- Child 1 closes fd[0] (doesn't read from pipe)
- Child 2 closes fd[1] (doesn't write to pipe)

Program 2 got: Hello from Program 1!

1. Programs can be self-contained, reading from stdin and write to stdout without needing to know about
   other programs. This means the same program can be reused in many different
   contexts and pipelines without modification.

2. The OS can redirect streams to files,
   pipes, and network sockets. This allows
